## 🚀 前端静态站点部署

### 1. Vercel（最推荐）⭐⭐⭐⭐⭐

**优点**：
- ✅ 免费套餐非常慷慨（100GB 带宽/月）
- ✅ 自动 HTTPS
- ✅ 全球 CDN 加速
- ✅ 支持自动部署（连接 GitHub）
- ✅ 自定义域名免费
- ✅ 零配置，上传即用

**限制**：
- ❌ 仅支持静态站点或 Serverless 函数
- ❌ 需要后端单独部署

**部署步骤**：
```bash
# 方式 1：使用 Vercel CLI
npm install -g vercel
cd frontend
vercel

# 方式 2：连接 GitHub（推荐）
# 1. 访问 vercel.com，注册账号
# 2. 点击 "Import Project"
# 3. 连接 GitHub 仓库
# 4. 选择 frontend 目录
# 5. 自动部署完成！
```

**访问地址**：`https://your-project.vercel.app`

---

### 2. Netlify ⭐⭐⭐⭐⭐

**优点**：
- ✅ 100GB 带宽/月（免费）
- ✅ 自动 HTTPS
- ✅ 全球 CDN
- ✅ 支持 Serverless 函数
- ✅ 表单处理功能
- ✅ 自动部署（GitHub/GitLab）

**部署步骤**：
```bash
# 方式 1：使用 Netlify CLI
npm install -g netlify-cli
cd frontend
netlify deploy --prod

# 方式 2：拖拽部署（最简单）
# 1. 访问 netlify.com
# 2. 将 frontend/dist 目录拖到页面
# 3. 立即获得访问地址！
```

**访问地址**：`https://your-project.netlify.app`

---

### 3. GitHub Pages ⭐⭐⭐⭐

**优点**：
- ✅ 完全免费
- ✅ 支持自定义域名
- ✅ 自动 HTTPS
- ✅ 与 GitHub 集成

**限制**：
- ❌ 仅支持静态站点
- ❌ 有流量限制（每月 100GB）
- ❌ 需要公开仓库（或 GitHub Pro）

**部署步骤**：
```bash
# 1. 创建 GitHub 仓库
# 2. 推送代码
git init
git add frontend/dist
git commit -m "Deploy"
git push

# 3. 在仓库设置中启用 GitHub Pages
# Settings → Pages → Source: /frontend/dist
```

**访问地址**：`https://your-username.github.io/repo-name`

---

### 4. Cloudflare Pages ⭐⭐⭐⭐⭐

**优点**：
- ✅ 无限带宽（免费！）
- ✅ 全球 CDN 加速
- ✅ 自动 HTTPS
- ✅ 支持 Serverless 函数
- ✅ 与 Cloudflare 生态集成

**部署步骤**：
```bash
# 方式 1：连接 GitHub
# 1. 访问 dash.cloudflare.com
# 2. Workers & Pages → Create Application → Pages
# 3. 连接 GitHub 仓库
# 4. 选择 frontend 目录，构建命令：npm run build
# 5. 自动部署

# 方式 2：使用 Wrangler CLI
npm install -g wrangler
cd frontend
wrangler pages deploy dist
```

**访问地址**：`https://your-project.pages.dev`

## 🔧 全栈应用部署

### 5. Railway ⭐⭐⭐⭐

**优点**：
- ✅ $5 免费额度/月（足够小型项目）
- ✅ 支持数据库（PostgreSQL, MySQL, Redis）
- ✅ 自动 HTTPS
- ✅ 环境变量管理
- ✅ Docker 支持
- ✅ 自动部署（GitHub）

**部署步骤**：
```bash
# 1. 访问 railway.app，注册账号
# 2. 点击 "New Project"
# 3. 选择 "Deploy from GitHub repo"
# 4. 选择仓库
# 5. 配置服务：
#    - 后端：选择 backend 目录，设置启动命令：npm start
#    - 前端：选择 frontend 目录，设置构建命令：npm run build
# 6. 自动部署！
```

**访问地址**：`https://your-project.railway.app`

---

### 6. Render ⭐⭐⭐⭐

**优点**：
- ✅ 免费套餐可用（有限制）
- ✅ 支持 PostgreSQL 数据库
- ✅ 自动 HTTPS
- ✅ 自动部署（GitHub）
- ✅ Docker 支持

**限制**：
- ⚠️ 免费版应用会休眠（15分钟无活动后）
- ⚠️ 休眠后首次访问需要等待启动（约30秒）

**部署步骤**：
```bash
# 1. 访问 render.com，注册账号
# 2. 点击 "New +" → "Web Service"
# 3. 连接 GitHub 仓库
# 4. 配置：
#    - Name: backend
#    - Root Directory: backend
#    - Build Command: npm install && npm run build
#    - Start Command: npm start
# 5. 重复步骤创建前端服务
```

**访问地址**：`https://your-project.onrender.com`

---

### 7. Fly.io ⭐⭐⭐

**优点**：
- ✅ 3个共享CPU应用免费
- ✅ 支持数据库
- ✅ 全球部署
- ✅ Docker 支持

**缺点**：
- ❌ 配置相对复杂
- ❌ 需要信用卡验证（但不收费）

**部署步骤**：
```bash
# 1. 安装 Fly CLI
curl -L https://fly.io/install.sh | sh

# 2. 登录
fly auth login

# 3. 在项目根目录初始化
fly launch

# 4. 配置并部署
fly deploy
```

## 🏠 自托管方案

### 10. 本地服务器 + DDNS

**需要的工具**：
- 动态 DNS 服务（如 DuckDNS, No-IP - 免费）
- 路由器端口转发
- 或使用 Cloudflare Tunnel

**步骤**：
```bash
# 1. 注册 DuckDNS（duckdns.org）
# 2. 创建域名，获取 Token
# 3. 配置路由器端口转发
# 4. 安装 DuckDNS 客户端更新 IP
# 5. 访问：http://your-domain.duckdns.org
```
