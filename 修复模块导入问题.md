# 修复模块导入问题

## 问题描述

启动后端服务时出现错误：
```
Error [ERR_MODULE_NOT_FOUND]: Cannot find module 'c:\low_coding_platform\backend\dist\routes\projects'
```

## 原因

Node.js ES 模块系统要求导入路径必须包含文件扩展名（`.js`），即使源文件是 TypeScript（`.ts`）。

## ✅ 已修复

已修复所有后端文件中的导入语句，添加了 `.js` 扩展名：

- ✅ `backend/src/index.ts` - 所有路由导入
- ✅ `backend/src/routes/*.ts` - 所有 prisma 和 service 导入

## 验证修复

运行以下命令验证：

```bash
cd backend
npm run build
npm start
```

应该看到：
```
Server is running on http://0.0.0.0:3001
✅ 数据库连接成功
```

## 如果仍有问题

1. **清理并重新构建**
   ```bash
   cd backend

   npm run build

   ```

2. **检查所有导入语句**
