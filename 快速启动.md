# 快速启动指南

## 当前问题
服务未启动，显示 ERR_CONNECTION_REFUSED 错误。

## 解决步骤

### 1. 检查依赖是否安装

打开终端，执行：

```bash
# 检查前端依赖
cd frontend
dir node_modules

# 检查后端依赖  
cd ..\backend
dir node_modules
```

如果 `node_modules` 文件夹不存在或为空，需要安装依赖。

### 2. 安装依赖

**方式一：使用启动脚本（最简单）**
- 双击运行 `启动项目.bat` 文件
- 脚本会自动安装依赖并启动服务

**方式二：手动安装**

在项目根目录执行：
```bash
npm install
```

或者分别安装：
```bash
cd frontend
npm install

cd ..\backend
npm install
```

### 3. 启动服务

**方式一：使用启动脚本**
- 双击 `启动项目.bat`

**方式二：手动启动**

打开两个终端窗口：

**终端1 - 启动后端：**
```bash
cd backend
npm run dev
```

**终端2 - 启动前端：**
```bash
cd frontend
npm run dev
```

### 4. 验证服务

等待10-15秒后，访问：
- 前端: http://localhost:3000
- 后端: http://localhost:3001/api/health

## 常见问题

### 依赖安装很慢？
- 首次安装需要下载大量包，可能需要5-10分钟
- 可以使用国内镜像加速：`npm install --registry=https://registry.npmmirror.com`

### 端口被占用？
- 检查是否有其他程序占用3000或3001端口
- 可以修改 `frontend/vite.config.ts` 和 `backend/src/index.ts` 中的端口号

### 仍然无法访问？
1. 检查终端窗口是否有错误信息
2. 确认服务已启动（应该看到 "Server is running" 或 "VITE" 相关信息）
3. 检查防火墙设置

## 推荐操作

**最简单的方式：直接双击 `启动项目.bat` 文件！**

脚本会自动处理所有步骤。
